【1. サービス概要】

サービス名: Shiretto Cat（仮）

コンセプト: ユーザーが撮影した日常風景やカフェの写真（マグカップや皿など）に、写真の主役を邪魔しない「しれっと潜む」ミニマルな白い線画の猫をAIで自動合成し、SNSでシェアできるサービス。

ターゲット: Instagram等のSNSで日常の写真をシェアする若い世代やカフェ愛好家。

【2. 技術要件（クロスプラットフォーム＆エッジAI）】
最大の要件は「サーバー代ゼロ（推論コストゼロ）」で実現すること、そして「Webとネイティブアプリ（iOS/Android）の両方でリリース可能」であることです。以下の技術スタックを前提に設計してください。

開発言語: TypeScript

UIフレームワーク: React

クロスプラットフォーム化: Capacitor (Reactで構築したWebアプリをネイティブアプリとしてラップし、カメラ機能等にアクセスする)

AI推論エンジン: WebGPUを活用したブラウザ内推論（Transformers.js や ONNX Runtime Web などを使用し、端末のGPUでローカル処理を行う）

【3. コア機能：AI画像処理パイプライン要件】
アプリ内で実行されるAI推論（セグメンテーションおよび画像生成）のシステムプロンプトおよび処理フローは、以下のルールを厳格に守って実装してください。

ステップA（汎用セグメンテーション）:
画像内のメイン容器（カップ、お皿など）、容器の持ち手、開口部（中身）、既存のロゴ/柄、およびテーブル面を正確に認識・分割すること。

ステップB（ネガティブアノテーション・禁止領域の除外）:
以下の領域には絶対に猫を生成（合成）しないよう、マスキング処理を実装すること。

容器の開口部（中）、飲み物・食べ物の真上。

容器のフチの真上。

容器に元からある柄やロゴの真上。

接地面のない空中の不自然な位置。

ステップC（ポジティブ指示・生成領域の決定）:
「容器の側面に沿ったテーブル面」や「容器の陰」を最適な配置場所として算出し、毛並み等のテクスチャを排除した「白い連続線のみのミニマルな猫（伫む、または覗くポーズ）」を生成して合成すること。

【4. 出力要求】
上記を踏まえ、以下の3点を提示してください。

システムアーキテクチャ概要: WebGPUを活用した画像処理フローと、Capacitorを用いたファイル/カメラアクセスの構成。

主要ライブラリ構成（package.jsonの要点）: 必要なnpmパッケージのリスト。

推論パイプラインの疑似コード: 画像を受け取り、マスキング領域を計算し、WebGPU経由で画像を生成・合成する処理の中核となるTypeScriptコードの設計思想。
