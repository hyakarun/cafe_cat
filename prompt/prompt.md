# Shiretto Cat（シレットキャット）開発プロンプト

【1. サービス概要】
サービス名: Shiretto Cat（仮）
コンセプト: ユーザーが撮影した日常風景やカフェの写真（マグカップや皿など）に、写真の主役を邪魔しない「しれっと潜む」ミニマルな白い線画の猫をAIで自動合成し、SNSでシェアできるサービス。
ターゲット: Instagram等のSNSで日常の写真をシェアする若い世代やカフェ愛好家。

【2. デザイン・UIに関する要件】
- **コンセプチュアルデザイン**: 写真（画像）を主役にした、温かみのあるモダンでプレミアムなデザイン（Photography-forward, warm aesthetic）。
- **Airbnbデザインシステムの採用**: 以下のアセット・トークンを厳格に使用すること。
  - Rausch Red（ラウシュレッド）: 主なアクセントカラー `#FF5A5F`
  - Near-Black（ニアブラック）: テキスト等の基本色 `#484848`
  - 3-layer shadows: 3層のシャドウによる奥行きの表現
  - Rounded corners: UIコンポーネントの角丸加工
- **リッチなユーザー体験**: ホバーエフェクトやトランジション時のダイナミックで気持ち良いマイクロアニメーションを導入すること。タイポグラフィは `Inter`、`Roboto`、`Outfit` などのモダンなGoogleフォントを使用。

【3. 技術要件（クロスプラットフォーム＆エッジAI）】
最大の要件は「サーバー代ゼロ（推論コストゼロ）」で実現すること、そして「Webとネイティブアプリ（iOS/Android）の両方でリリース可能」であることです。以下の技術スタックを前提に設計・実装してください。
- 開発言語: TypeScript
- UIフレームワーク: React
- バンドラー・ビルドツール: Vite
- クロスプラットフォーム化: Capacitor (Reactで構築したWebアプリをネイティブアプリとしてラップし、カメラ機能等にアクセスする)
- AI推論エンジン: WebGPUを活用したブラウザ内推論（Transformers.js や ONNX Runtime Web などを使用し、端末のGPUでローカル処理を行う）

【4. コア機能：AI画像処理パイプライン要件】
アプリ内で実行されるAI推論（セグメンテーションおよび画像生成）のフローは以下のルールを守ること。
- **ステップA（汎用セグメンテーション）**: 画像内のメイン容器（カップ、お皿など）、持ち手、開口部（中身）、既存のロゴ/柄、およびテーブル面を正確に認識・分割する。
- **ステップB（禁止領域のマスキング）**: 「容器の中や真上」「フチの真上」「元からあるロゴの真上」「接地面のない空中の不自然な位置」には絶対に猫を生成しないこと。
- **ステップC（ポジティブ指示）**: 「容器の側面に沿ったテーブル面」や「容器の陰」を最適な配置場所として算出し、毛並み等のテクスチャを排除した「白い連続線のみのミニマルな猫」を合成する。
- **直近の技術課題**: Viteとのバンドル競合やモデルの読み込みに起因する初期化エラー（`registerBackend` や `Unauthorized access` など）を解決し、パイプラインを正常に動作させること。

【5. AIアシスタント コーディングガイドライン（厳守ルール）】
AIは、以下のユーザー独自ルールに厳格に従って開発を進めること。
1. **言語の統一**: ユーザーへの返答、思考プロセス（プラン）、提案、タスクリスト等のファイルを含め、すべて **日本語** で行うこと。
2. **Gitの継続的運用**: 実装が一段落（機能の区切り）するごとに、必ず `git push` まで行うこと。コミットメッセージを作成する際は、前回からの差分等を確認して適切なコメントを入れること。
3. **ファイル分割の徹底**: ロジックやコンポーネントは一つのファイルに肥大化させず、可能な限りファイルを分割しつつ開発を進めること。
4. **常時リファクタリング**: 新しい機能を追加するだけでなく、つねにコードのリファクタリングを意識しながら開発を進めること。
